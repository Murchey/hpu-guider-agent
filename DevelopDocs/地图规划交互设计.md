# 地图规划交互设计

## 1. 页面入口与触发机制

地图规划页面位于"AI 对话"导航栏标签右侧，单独点击仅显示空地图。

触发显示路线数据的流程：
1. AI 对话气泡底部包含 `[MAP_DATA]{JSON}[/MAP_DATA]` 标签
2. 前端检测到该标签后，在气泡底部显示"转到地图"按钮
3. 用户点击按钮，跳转至地图规划页面，前端解析 JSON 并在地图上绘制路线

---

## 2. 数据格式规范

### 2.1 itinerary 字段定义

```json
{
  "itinerary": [
    {
      "id": "start_01",
      "name": "洛阳龙门站",
      "type": "departure",
      "coordinates": { "lng": 112.463, "lat": 34.593 },
      "suggested_duration": 0,
      "travel_mode": "driving",
      "description": "行程起始点"
    }
  ]
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | String | 唯一标识，如 `spot_01` |
| `name` | String | 地点名称 |
| `type` | String | `departure` / `attraction` / `restaurant` / `hotel` |
| `coordinates` | Object | 经纬度（GCJ-02 坐标系） |
| `suggested_duration` | Number | 建议停留时长（分钟），出发地为 0 |
| `travel_mode` | String | 从此点到下一点的出行方式 |
| `description` | String | 地点简要介绍 |

`travel_mode` 可选值：`driving`（驾车）、`walking`（步行）、`transit`（公共交通）、`cycling`（骑行）

> **约束**：`itinerary` 数组的第一个元素必须为 `type: "departure"`。

## 3. 交互设计

### 3.1 拖拽重算

1. **固定起始点**：出发地卡片始终锁定在第一位，不可被拖走
2. **出行方式切换**：用户可手动更改每段行程的出行方式，前端即时重绘路径
3. **拖拽触发重算**：用户调整卡片顺序后，前端提取新坐标序列，调用高德 API 重新计算各段路径及耗时
4. **最终确认**：用户点击"确认行程"按钮后，将最终数组发送至后端存档或生成分享文案

### 3.2 视觉优化

- **拖拽占位**：拖拽过程中原位置显示虚线框（Ghost class），配合平滑过渡动画
- **交通耗时提示**：两张卡片间隙处显示交通方式图标与耗时，拖拽后数字动态跳变

## 4. 高德 API 集成

### 4.1 初始化与插件加载

安装加载器：
```bash
npm i @amap/amap-jsapi-loader --save
```

必须加载的插件：
- `AMap.Driving` — 驾车路径规划
- `AMap.Walking` — 步行路径规划
- `AMap.Transfer` / `AMap.Transit` — 公共交通路径规划（优先使用 `AMap.Transfer`，支持跨城）
- `AMap.Cycling` — 骑行路径规划

地图初始化：
```javascript
import AMapLoader from '@amap/amap-jsapi-loader'

const AMap = await AMapLoader.load({
  key: '<AMAP_KEY>',
  version: '2.0',
  plugins: ['AMap.Driving', 'AMap.Walking', 'AMap.Transfer', 'AMap.Cycling']
})

const map = new AMap.Map('map-container', {
  center: [116.397, 39.908],
  zoom: 12
})
```

### 4.2 路径样式规范

| 出行方式 | 颜色 | 色值 | 线型 |
|----------|------|------|------|
| 驾车 | 鲜蓝 | `#0084FF` | 实线 |
| 步行 | 青绿 | `#13C2C2` | 虚线 |
| 公交/地铁 | 紫色 | `#722ED1` | 实线 |
| 骑行 | 橙色 | `#FA8C16` | 实线 |
| 火车/高铁 | 红色 | `#F5222D` | 虚线弧线 |

### 4.3 多模式路径绘制

绘制流程：
1. 循环遍历 `itinerary` 数组，根据每段的 `travel_mode` 调用对应插件的 `search` 方法
2. 手动创建 `AMap.Polyline`（而非使用插件自带渲染），以支持多颜色混合显示
3. 绘制前先清理地图上已有的所有 `Polyline` 覆盖物
4. 绘制完成后调用 `map.setFitView()` 自动缩放视野

### 4.4 公共交通路径解析

`AMap.Transfer` / `AMap.Transit` 返回的数据结构为 **方案 → 路段 → 交通工具/步行**。

关键数据路径：
- `result.plans[0]` — 第一条推荐方案
- `plan.segments` — 路段数组
- `seg.transit.lines[0].path` — 城市内公交/地铁轨迹坐标
- `seg.walking.path` — 步行轨迹坐标
- `seg.railway.path` — 火车轨迹坐标（若有）
- `seg.railway.via_stops` — 途经站列表

分层绘制策略：
- **步行路段**：灰色虚线（`#999999`，`strokeStyle: 'dashed'`）
- **公交/地铁路段**：根据 `segment.transit.mode` 区分颜色
- **站点标记**：在上车点添加 `AMap.Marker`

### 4.5 火车路线处理

高德 API 返回的跨城火车方案通常不包含铁轨详细坐标。采用 **模拟弧线（Arc Path）** 方案生成可视化路径。

```javascript
function getArcPath(start, end, steps = 50) {
  const path = []
  const [startLng, startLat] = start
  const [endLng, endLat] = end
  const distance = Math.sqrt(
    Math.pow(endLng - startLng, 2) + Math.pow(endLat - startLat, 2)
  )
  const curveHeight = distance * 0.15

  for (let i = 0; i <= steps; i++) {
    const ratio = i / steps
    let lng = startLng + (endLng - startLng) * ratio
    let lat = startLat + (endLat - startLat) * ratio
    lat += Math.sin(ratio * Math.PI) * curveHeight
    path.push([lng, lat])
  }
  return path
}
```

渲染规范：
- 线型：虚线或加粗实线
- 颜色：红色 `#F5222D`
- 在弧线中点添加 `AMap.Text` 标签，显示车次（如 "G1812"）
- 点击轨迹弹出信息窗体，显示车次、历时及途经站数

### 4.6 降级策略

当路径规划返回 `NO_DATA` 或失败时，按以下顺序降级：
1. 公交规划失败 → 尝试驾车规划
2. 驾车规划失败 → 绘制起终点之间的直线

## 5. AI 提示词规范

### 5.1 触发场景

当用户表达以下意图时触发地图数据生成：
- "帮我规划一下去北京的路线"
- "这几个景点怎么走比较顺路？"
- "我想去故宫、长城和天安门，怎么安排？"

### 5.2 输出规则

1. 推荐超过 2 个地点时，必须在回复末尾附带 `[MAP_DATA]` JSON 块
2. `itinerary` 第一个元素必须为出发地，`type` 为 `departure`
3. 坐标系必须使用 **GCJ-02**（高德地图坐标系）
4. `[MAP_DATA]` 标签内部只能包含合法 JSON
5. 为每个地点指定 `travel_mode`（从该点到下一点的推荐方式）

### 5.3 示例

```
[MAP_DATA]{
  "itinerary": [
    {
      "id": "start_01",
      "name": "洛阳龙门站",
      "type": "departure",
      "coordinates": { "lng": 112.463, "lat": 34.593 },
      "suggested_duration": 0,
      "travel_mode": "transit",
      "description": "行程起始点"
    },
    {
      "id": "spot_01",
      "name": "龙门石窟",
      "type": "attraction",
      "coordinates": { "lng": 112.479, "lat": 34.556 },
      "suggested_duration": 180,
      "travel_mode": "transit",
      "description": "中国石刻艺术宝库"
    },
    {
      "id": "spot_02",
      "name": "夫子庙-秦淮风光带",
      "type": "attraction",
      "coordinates": { "lng": 118.789, "lat": 32.020 },
      "suggested_duration": 180,
      "travel_mode": "transit",
      "description": "南京地标"
    }
  ]
}[/MAP_DATA]
```

### 5.4 跨城公交 Web 服务 API

当 JS API 的 `AMap.Transfer` 无法返回跨城火车方案时，可通过 Web 服务 API 补充：

```
GET https://restapi.amap.com/v3/direction/transit/integrated
  ?key=<Web服务Key>
  &origin=112.463,34.593
  &destination=118.789,32.020
  &city=洛阳市
  &cityd=南京市
  &extensions=all
```

返回数据中 `segments[].transit.mode` 可能为 `TRAIN`（火车）、`SUBWAY`（地铁）、`BUS`（公交）、`WALK`（步行）。需针对不同模式分别解析轨迹并绘制。
