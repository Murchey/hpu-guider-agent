# AI对话气泡检测机制
检测到AI对话气泡底部含有[MAP_DATA]{JSON格式数据}[/MAP_DATA]时，给AI气泡底部增加一个“转到地图”按钮，点击后跳转到地图规划页面。地图规划页面会根据JSON格式数据绘制出地图，并在地图上显示出用户的位置。

前端根据经纬度在地图组件上绘制地图。要求 `itinerary` 数组的第一个元素必须是 `type: "departure"`。`travel_mode` 决定从**当前地点到下一个地点**的出行方式。

## 交互设计
“拖拽重算”的实现逻辑（重点）
1. **固定起始点**：用户无法将其他地点拖动到 "出发地" 之前，确保行程始终从出发地开始。
2. **出行方式选择**：用户可以手动更改每一段行程的出行方式（步行、公交、驾车、骑行），前端即时重绘路径。
3. **初始加载**： 用户输入需求，请求 Coze。Coze 返回包含出发地及景点的 JSON。
3. **前端渲染**： Vue3 拿到 JSON，左侧用 vue-draggable-plus 渲染，右侧在高德地图上打点画线。

用户拖拽： 用户觉得先去景点 C 再去景点 B 更好，将 C 的卡片拖到了 B 前面。

前端本地重算（无缝体验）： * vue-draggable-plus 瞬间改变了数组顺序。

Vue3 监听到数组变化，立刻提取新的经纬度顺序。

前端直接发送请求给高德地图 API：“请告诉我从 C 到 B 的打车需要多久？”

高德 API 几百毫秒内返回数据，前端立刻更新卡片之间的“交通时间”提示。

最终确认： 用户拖拽调整满意后，点击“确认行程”按钮。前端将最终的数组顺序发回给后端的 Node.js 进行存档，或者再次发给 Coze 让它根据最终路线生成一份“分享文案”。

## 视觉与交互体验优化
占位符与动画： 在拖拽过程中，利用 CSS 给原本的位置留出一个“虚线框”（Ghost class），并增加平滑的过渡动画，让拖拽手感更“粘滞”、更高级。

交通耗时悬浮提示： 在两张卡片的间隙处，设计一个带有小汽车（Van）图标的模块。拖拽顺序改变后，这里数字（如：打车 15 分钟）要有动态跳变的数字效果。

# 高德API使用
在 Vue 3 中调用高德地图（AMap）API 进行路线绘制，最优雅的方式是使用官方提供的 `@amap/amap-jsapi-loader`。针对你提到的“拖拽式规划”，核心逻辑是：监听前端列表顺序的变化，提取坐标点数组，然后调用高德的路径规划插件进行重新渲染。

实现步骤
1. **安装加载器**：
   `npm i @amap/amap-jsapi-loader --save`

2. **初始化地图与插件**：
   在 Vue 组件中引入并初始化地图，同时加载 `AMap.Driving`（驾车）插件。

3. **调用路径规划服务**：
   使用插件的 `search` 方法，传入起始点、终点及途经点（Waypoints）。对于分段耗时，通过循环调用两点间的 `search` 来获取。

## 核心代码实现逻辑
1. **地图初始化**
```javascript
import AMapLoader from '@amap/amap-jsapi-loader';

let map = null;
let driving = null;

AMapLoader.load({
    key: "你的高德Key",
    version: "2.0",
    plugins: ['AMap.Driving']
}).then((AMap) => {
    map = new AMap.Map("map-container", { center: [116.397, 39.908], zoom: 12 });
    driving = new AMap.Driving({
        map: map,
        hideMarkers: true // 隐藏默认标记，由前端自定义渲染序号标记
    });
});
```

2. **根据拖拽后的列表重绘路线与计算耗时**
```javascript
const drawRoute = () => {
    if (!driving || itineraryList.value.length < 2) return;

    const start = [itineraryList.value[0].lng, itineraryList.value[0].lat]; 
    const end = [itineraryList.value[itineraryList.value.length - 1].lng, itineraryList.value[itineraryList.value.length - 1].lat];
    const waypoints = itineraryList.value.slice(1, -1).map(item => [item.lng, item.lat]);

    driving.search(start, end, { waypoints }, (status, result) => {
        if (status === 'complete') {
            // 路线绘制成功后，分段计算每两个点之间的时间
            calculateSegmentTimes();
        }
    });
};
```

## 3. 公共交通 (Transit) 进阶实现

规划公共交通（公交/地铁）的路线并绘制轨迹，比驾车或步行要复杂一些。

### 3.1 核心难点
驾车路线通常是一条连续的线，而公交路线是“分段”的（步行 -> 坐车 -> 步行 -> 换乘 -> 坐车）。因此，在 Vue 3 中调用高德 API 绘制公交轨迹，不能只画一条线，而是需要组合绘制（步行用虚线，地铁/公交用实线，并添加站点标记）。

### 3.2 🚀 核心思路
1.  **调用插件**：使用 `AMap.Transit` 插件进行路径规划。
2.  **解析结果**：API 返回的数据结构是 `方案 -> 路段 -> 交通工具/步行`。
3.  **分层绘制**：
    *   **步行路段**：绘制为灰色虚线。
    *   **公交/地铁路段**：绘制为彩色实线（地铁通常用特定颜色）。
    *   **站点**：添加标记点（Marker）表示上车点和下车点。

### 3.3 💻 示例代码实现

```vue
<template>
  <div id="container" style="width: 100%; height: 600px;"></div>
  <button @click="planTransitRoute">规划公交路线</button>
</template>

<script setup>
import { onMounted } from 'vue';
import AMapLoader from '@amap/amap-jsapi-loader';

let map = null;
let AMap = null;

// 1. 初始化地图
onMounted(async () => {
  AMap = await AMapLoader.load({
    key: "你的Key",
    version: "2.0",
    plugins: ["AMap.Transit"] // 必须加载公交规划插件
  });

  map = new AMap.Map('container', {
    zoom: 13,
    center: [116.480975, 39.989615] // 默认中心点
  });
});

// 2. 规划并绘制路线
const planTransitRoute = () => {
  const start = [116.397428, 39.90923]; // 起点
  const end = [116.480975, 39.989615];  // 终点

  // 创建公交规划实例
  const transit = new AMap.Transit({
    map: map,
    hideMarkers: false, // 是否隐藏默认起终点标记
    policy: AMap.TransitPolicy.LEAST_TIME, // 策略：最快捷
  });

  // 搜索路线
  transit.search(start, end, (status, result) => {
    if (status === 'complete') {
      console.log('公交规划成功', result);
      drawTransitPath(result.plans[0]); // 绘制第一条推荐方案
    } else {
      console.error('公交规划失败');
    }
  });
};

// 3. 自定义绘制逻辑 (关键步骤)
const drawTransitPath = (transitData) => {
  const segments = transitData.segments; // 获取所有路段

  segments.forEach((segment) => {
    const path = segment.transit ? segment.transit.path : segment.walking.path;
    const isWalk = !!segment.walking;

    if (isWalk) {
      // --- 绘制步行路段 ---
      new AMap.Polyline({
        path: path,
        strokeColor: '#999',    // 灰色
        strokeStyle: 'dashed',  // 虚线
        strokeWeight: 3,
        strokeOpacity: 0.6,
        map: map
      });
    } else {
      // --- 绘制公交/地铁路段 ---
      const mode = segment.transit.mode;
      
      // 1. 画线
      new AMap.Polyline({
        path: path,
        strokeColor: mode === 'SUBWAY' ? '#9013FE' : '#0066FF', // 地铁紫色，公交蓝色
        strokeWeight: 5,
        map: map
      });

      // 2. 添加站点标记 (上车点/下车点)
      if (segment.departure_stop) {
        new AMap.Marker({
          position: segment.departure_stop.location,
          title: segment.departure_stop.name,
          map: map,
          icon: 'https://a.amap.com/jsapi_demos/static/demo-center/icons/poi-marker-default.png'
        });
      }
    }
  });
};
</script>
```

### 3.4 火车路线规划方案

在高德地图 JS API 中，跨城公交规划（`AMap.Transfer`）虽然可以处理火车/高铁方案，但往往不提供铁轨的详细地理轨迹坐标。为了在视觉上提供更好的用户体验，我们采用“模拟弧线”方案。

#### 🧐 1. 核心挑战
*   **数据缺失**：API 返回起止站点信息和车次，但通常不提供完整的经纬度轨迹数组。
*   **视觉生硬**：直接用直线连接两个城市会显得非常生硬，不符合航空或高铁的视觉直觉。

#### 🛠️ 2. 解决方案：模拟弧形路径 (Arc Path)
我们通过数学插值算法，在起点站和终点站之间生成一条向北侧微凸的“大圆弧”路径，模拟真实航线效果。

**Vue 实现代码（计算弧线坐标）：**
```javascript
/**
 * 获取两个坐标点之间的弧形路径（模拟高铁/航线）
 * @param {Array} start - [lng, lat] 起点
 * @param {Array} end - [lng, lat] 终点
 * @param {Number} steps - 弧线的分段数量
 */
function getArcPath(start, end, steps = 50) {
  const path = [];
  const [startLng, startLat] = start;
  const [endLng, endLat] = end;

  // 计算中点和距离
  const distance = Math.sqrt(Math.pow(endLng - startLng, 2) + Math.pow(endLat - startLat, 2));
  const curveHeight = distance * 0.15; // 弧度高度系数，可根据审美调整

  for (let i = 0; i <= steps; i++) {
    const ratio = i / steps;
    let lng = startLng + (endLng - startLng) * ratio;
    let lat = startLat + (endLat - startLat) * ratio;
    
    // 添加正弦波偏移，模拟弧度
    lat += Math.sin(ratio * Math.PI) * curveHeight; 
    path.push([lng, lat]);
  }
  return path;
}
```

#### 🎨 3. 渲染规范
*   **线型**：使用 `dashed`（虚线）或加粗的实线。
*   **颜色**：统一使用红色 (`#F5222D`) 表示轨道交通。
*   **信息辅助**：在弧线中点位置添加 `AMap.Text` 标签，显示车次（如 "G1812"）。

### 3.5 🔍 关键数据结构解析说明
在绘制函数中，需兼容以下数据路径：
*   `plan.segments`: 路段数组。
*   `seg.transit.lines[0].path`: 城市内公交/地铁轨迹。
*   `seg.railway.path`: 火车轨迹（若有）。
*   `seg.railway.via_stops`: 途径站（用于在轨迹缺失时进行连接）。

### 3.6 进阶优化建议
1.  **信息窗体**：点击轨迹显示详细车次、历时及票价（若 API 支持）。
2.  **视野自适应**：绘图后调用 `map.setFitView()`，并增加 `padding` 避免长途路径贴边。
3.  **模式兜底**：若公交规划返回 `NO_DATA`，自动切换至驾车模式或直线连接。

在 `drawTransitPath` 函数中，你需要理解高德返回的数据结构：
*   `result.plans`: 这是一个数组，包含多个推荐方案。通常我们取 `plans[0]` 绘制第一条。
*   `transit.segments`: 一个完整的公交方案由多个“路段”组成。
    *   路段 1：可能是从家 **步行** 到地铁站。
    *   路段 2：乘坐 **地铁** 10号线。
    *   路段 3：**步行** 到公司。
*   `segment.transit`: 如果该路段是公交/地铁，则包含此对象。
*   `segment.walking`: 如果该路段是步行，则包含此对象。

### 3.5 🎨 进阶优化建议
1.  **添加信息窗体**：在点击地铁/公交线段时，显示线路名称（如“地铁10号线”）和经过站数。
2.  **调整地图视野**：绘制完成后，调用 `map.setFitView()` 自动缩放。
3.  **处理跨城公交**：如果涉及跨城，API 会返回包含火车（RAILWAY）的路段，需针对 `railway` 模式设置颜色。
## 3. 多模式路径绘制实现 (Web 端)
针对不同的 `travel_mode`，前端需要调用高德地图不同的插件进行路径规划和渲染。

### 3.1 核心插件加载
在初始化地图时，必须显式加载以下插件：
- `AMap.Driving`: 驾车路径规划
- `AMap.Transit`: 公共交通路径规划
- `AMap.Walking`: 步行路径规划
- `AMap.Cycling`: 骑行路径规划

### 3.2 样式区分规范
为了增强视觉辨识度，不同出行方式应使用不同的路径颜色（采用高对比度方案）：
- **驾车**: 鲜蓝色 (`#0084FF`)
- **步行**: 青绿色 (`#13C2C2`)
- **公交**: 紫色 (`#722ED1`)
- **骑行**: 橙色 (`#FA8C16`)
- **火车**: 红色 (`#F5222D`)
- **步行(换乘)**: 灰色虚线 (`#999999`)

### 3.3 实现逻辑说明
由于行程是分段的（例如：第一段驾车，第二段步行），前端需要：
1. **分段请求**：循环行程数组，根据当前点到下一点的 `travel_mode` 调用对应插件的 `search` 方法。
2. **手动绘线**：为了支持多颜色混合显示，不直接使用插件的默认渲染（`map: map`），而是获取规划结果中的 `path` 坐标数组，手动创建 `AMap.Polyline` 实例并设置对应颜色。
3. **公共交通 (Transit) 特殊处理**：
   - `AMap.Transit` 返回的是 `plans` 数组，通常取第一个 `plans[0]`。
   - `plans[0]` 不直接提供完整 `path`，需遍历其 `segments`。
   - 每个 `segment` 可能包含 `transit.path`（公交/地铁）或 `walking.path`（步行换乘），必须将它们**拼接**起来才能得到完整轨迹。
4. **清理机制**：在重排行程或更改模式时，需先清理地图上已有的所有 `Polyline` 覆盖物。
