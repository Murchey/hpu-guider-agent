# AI 交互标签协议（选项卡）

AI 可通过发送特定格式的标签来触发前端的交互弹窗（单选卡和多选卡）。

---

## 1. 单选卡（场景/方案选择）

用于提供 1-3 个固定的方案供用户选择。

### 标签格式

使用 `[SCENE_DATA]` 标签包裹 JSON 数据。

### JSON 结构

```json
{
  "description": "方案的简要说明，显示在按钮上方",
  "btn1": "按钮1文字",
  "btn2": "按钮2文字",
  "btn3": "按钮3文字"
}
```

- `description`：可选，字符串类型
- 字段名必须为 `btn1`、`btn2`、`btn3`（全部小写）
- 只需 1-2 个按钮时，多余字段设为空字符串 `""` 或省略

### 示例

```
[SCENE_DATA]{"description": "以下是为您准备的三个行程方向：", "btn1": "查看深度文化方案", "btn2": "查看休闲漫步方案", "btn3": "查看美食打卡方案"}[/SCENE_DATA]
```

---

## 2. 多选卡（景点/选项列表）

### 标签格式

使用 `[SELECT_DATA]` 标签包裹 JSON 数据。

### JSON 结构

```json
{
  "description": "景点的简要介绍",
  "options": ["选项1", "选项2", "选项3"]
}
```

- `description`：可选，字符串类型
- `options`：数组类型，建议不超过 15 个

### 示例

```
[SELECT_DATA]{"description": "洛阳最受欢迎的景点：", "options": ["龙门石窟", "白马寺", "洛邑古城", "九州池"]}[/SELECT_DATA]
```

---

## 3. 用户反馈机制 (USER_CHOICE)

用户在弹窗中做出选择后，前端**自动隐藏发送**格式化指令给 AI：

- **单选反馈**：`model:景点推荐\n[USER_CHOICE]{"value": "所选按钮文字"}[/USER_CHOICE]`
- **多选反馈**：`model:确定游览方案\n[USER_CHOICE]{"values": ["选项1", "选项2"]}[/USER_CHOICE]`

AI 收到 `[USER_CHOICE]` 后，应结合上下文（用户画像等）直接生成对应方案。

---

## 4. 调用规范

| 规则 | 说明 |
|------|------|
| JSON 标准 | 必须使用双引号，严禁单引号 |
| 多行文本 | 使用 `\n` 转义换行 |
| 标签位置 | 放在回复内容的**最后一行** |
| 内容清理 | 前端解析成功后自动隐藏 `[...]...[/...]` 部分 |
| 互斥性 | 同一次回复中只调用一种卡片类型 |
