# 文途智行 项目流程与技术综述

## 1. 项目定位

"文途智行"是基于 Vue 3 + 大语言模型（LLM）的智慧导游助手，解决传统导览工具"信息碎片化"、"交互单向性"和"缺乏个性化"的问题。

核心能力：
- **个性化规划**：通过首页问卷（人数、天数、预算、风格等）生成针对性行程
- **深度文化交互**：AI 模拟专业导游口吻，结合历史背景进行解说
- **结构化交互闭环**：自定义 JSON 标签协议（`[SCENE_DATA]`、`[SELECT_DATA]`），将 AI 输出转化为前端 UI 组件
- **多模态感知**：支持图片上传与识别，现场拍照即可获得解说

---

## 2. 技术栈

### 2.1 核心框架
- **Vue 3**（Composition API + `<script setup>`）
- **TypeScript**
- **Vite 5**（构建与 HMR）

### 2.2 UI 与渲染
- **Element Plus**：组件库，深度定制样式
- **Markdown-it**：AI 输出的 Markdown 渲染，配置 `linkify` 和 `breaks` 插件
- **视差滚动 (Parallax)**：监听鼠标坐标动态计算背景偏移
- **碎片飞散 (Shatter)**：DOM 碎片网格化模拟物理破碎效果

### 2.3 网络与协议
- **Axios**：RESTful 请求（如 Coze 文件上传）
- **Fetch API + SSE**：流式传输，实现"打字机"实时输出

---

## 3. 核心功能模块

### 3.1 首页 (IndexPage.vue)
- 用户画像问卷：10+ 维度（出发日期、交通方式、风格偏好等），数据存入 `localStorage`
- 动态视觉交互：多层视差背景 + 碎片飞散特效

### 3.2 对话中心 (AiDialoguePage.vue)
- **多模态输入**：图片上传自动切换 Coze（`file_id`）或 Base64 模式
- **工作模式切换**：`el-segmented` 分段控制器在"互动问答"与"景点介绍"间切换
- **消息渲染**：`formatAssistantMessage` 处理 Markdown + 自定义标签提取
- **对话持久化**：Pinia store (`chatStore.ts`) + localStorage 实现会话保存/切换/删除/重命名
- **消息交互**：复制、修改（截断后续对话并重新发送）、删除

### 3.3 地图规划 (MapPlanningPage.vue)
- 高德地图 SDK 集成（`@amap/amap-jsapi-loader`）
- 多模式路径绘制：驾车/步行/公交/骑行/火车，不同颜色与线型
- 跨城火车路线：Web 服务 API + `getArcPath` 模拟弧线
- 拖拽排序：`vue-draggable-plus` 调整顺序后自动重绘
- APP 唤起：`amapuri://` 协议 + iframe 静默尝试

### 3.4 设置中心 (SettingPage.vue)
- API 动态热切换（Key、Base URL、模型 ID）
- System Prompt 注入与管理

## 4. 自定义交互协议

### 4.1 单选卡 `[SCENE_DATA]`
```
[SCENE_DATA]{"description": "...", "btn1": "...", "btn2": "...", "btn3": "..."}[/SCENE_DATA]
```

### 4.2 多选卡 `[SELECT_DATA]`
```
[SELECT_DATA]{"description": "...", "options": ["选项1", "选项2"]}[/SELECT_DATA]
```

### 4.3 静默反馈 `[USER_CHOICE]`
用户在弹窗做出选择后，前端通过 `handleSendHidden` 发送隐藏指令，对用户透明：
```
[USER_CHOICE]{"value": "..."}[/USER_CHOICE]
```

## 5. 业务流程

### 阶段一：画像感知
用户填写问卷 → 存入 localStorage → 跳转对话页 → 静默发送 `model:用户画像`

### 阶段二：意图确认
AI 回复欢迎语 + `[SCENE_DATA]` → 用户点击按钮 → 静默发送 `[USER_CHOICE]`

### 阶段三：景点筛选
AI 推荐景点 + `[SELECT_DATA]` → 用户勾选 → 静默发送 `[USER_CHOICE]`

### 阶段四：方案生成
AI 生成完整行程（交通+时间+避坑提示），用户可上传照片触发现场讲解

## 6. 关键技术细节

### 6.1 SSE 流式缓冲区管理

网络波动可能导致 JSON 截断，通过 buffer 拼接解决：

```typescript
buffer += decoder.decode(value, { stream: true })
const lines = buffer.split('\n')
buffer = lines.pop() || ''  // 最后一行可能不完整，留到下次处理
```

### 6.2 移动端响应式

- `computed` 根据 `isMobile` 动态返回弹窗宽度和字体
- 移动端输入框 `autosize` 限制行数，弹窗 `append-to-body` 避免键盘遮挡

### 6.3 提示词工程

- **少样本提示 (Few-shot)**：Prompt 中给出标签示例，引导 AI 生成规范 JSON
- **思维链 (CoT)**：按"读取画像 → 分析需求 → 提出建议"步骤思考
- **角色约束**：AI 称呼固定为"文途智行"，语气亲切专业
