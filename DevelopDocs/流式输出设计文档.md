# 流式输出 (SSE) 实现文档

## 1. 概述

Coze API 通过 **Server-Sent Events (SSE)** 协议实现流式输出。在调用 v3 版本的"发起对话 (Chat)" API 时，将请求体中的 `stream` 参数设置为 `true` 即可开启。

核心流程：
1. 发送 POST 请求至 `/v3/chat`，设置 `stream: true`
2. 服务器以 SSE 格式持续推送事件（`event:` + `data:`）
3. 前端逐行解析，将 `data` 中的字符片段追加到页面
4. 收到完成信号后结束渲染

## 2. API 调用方式

### 2.1 HTTP API (Fetch)

```bash
curl --location --request POST 'https://api.coze.cn/v3/chat' \
--header 'Authorization: Bearer <PAT_访问令牌>' \
--header 'Content-Type: application/json' \
--data-raw '{
    "bot_id": "<BOT_ID>",
    "user_id": "<用户标识>",
    "stream": true,
    "additional_messages": [
        {
            "role": "user",
            "content": "你好，请给我讲个笑话",
            "content_type": "text"
        }
    ]
}'
```

返回格式为 SSE 文本流，每条消息包含 `event:` 和 `data:` 两部分：
- `event: conversation.message.delta` — 消息增量事件
- `data: {"content": "片段文本"}` — 具体的字符片段

### 2.2 Node.js SDK

```bash
npm install @coze/api
```

```javascript
import { CozeAPI, COZE_CN_BASE_URL } from '@coze/api'

const client = new CozeAPI({
  token: '<PAT_访问令牌>',
  baseURL: COZE_CN_BASE_URL,
})

async function chatWithStream() {
  const stream = await client.chat.stream({
    bot_id: '<BOT_ID>',
    user_id: '<用户标识>',
    additional_messages: [
      {
        role: 'user',
        content: '你好，介绍一下你自己',
        content_type: 'text',
      },
    ],
  })

  for await (const part of stream) {
    if (part.event === 'conversation.message.delta') {
      process.stdout.write(part.data.content)
    }
  }
}
```

## 3. 前端 SSE 解析实现

### 3.1 基础解析逻辑

```typescript
const response = await fetch(url, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ stream: true, ... }),
  signal,
})

const reader = response.body.getReader()
const decoder = new TextDecoder()
let buffer = ''
let aiContent = ''

while (true) {
  const { done, value } = await reader.read()
  if (done) break

  buffer += decoder.decode(value, { stream: true })
  const lines = buffer.split('\n')
  buffer = lines.pop() || ''  // 保留未完成的最后一行

  for (const line of lines) {
    const trimmed = line.trim()
    if (!trimmed || !trimmed.startsWith('data:')) continue

    const dataStr = trimmed.slice(5).trim()
    if (dataStr === '[DONE]') continue

    try {
      const data = JSON.parse(dataStr)
      if (data.content) {
        aiContent += data.content
        // 更新视图...
      }
    } catch (e) {
      console.warn('JSON 解析失败:', dataStr)
    }
  }
}
```

### 3.2 关键注意事项

| 问题 | 说明 |
|------|------|
| **JSON 截断** | 网络波动可能将一个完整的 `data: {...}` 拆分成多次推送。通过 `buffer` + `lines.pop()` 保留未完成行，下次循环拼接 |
| **非流式兜底** | 部分后端不支持流式，`reader.read()` 一次性返回完整 JSON 并 `done: true`。需在流结束后检查 buffer 中是否有未解析的 JSON |
| **多字段兼容** | 不同服务商返回结构不同，需兼容 `data.content`、`data.message.content`、`data.choices[0].delta.content` 等多种路径 |
| **AbortError** | 使用 `AbortController` 中断请求时，`fetch` 会抛出 `AbortError`，需在 catch 中忽略该异常 |

## 4. 事件类型参考

| 事件名 | 含义 |
|--------|------|
| `conversation.message.delta` | 消息增量（实时字符片段） |
| `conversation.message.completed` | 消息生成完成（包含完整内容） |
| `conversation.chat.completed` | 整轮对话完成 |
| `error` | 发生错误 |
