根据 Coze 的官方文档（docs.coze.cn），Coze API 主要是通过 **Server-Sent Events (SSE)** 协议来实现流式输出的。
### 概略介绍解决方案
要在 Coze 中开启流式输出，核心逻辑非常简单：
 1. **API 参数配置**：在调用 v3 版本的“发起对话 (Chat)” API 时，只需在请求体 (JSON) 中将 stream 参数设置为 true。
 2. **处理 SSE 响应**：Coze 服务器会以数据流（Stream）的形式持续向你的前端或后端发送事件（Events），你需要监听这些事件并逐步拼接文本，直到接收到对话完成的信号。
 3. **官方 SDK 支持**：如果你在 Node.js 环境或后端服务中集成，Coze 提供了官方 SDK，其中封装了专用的 .stream() 方法，使得处理流式数据变得极其简单。
### 展示代码和实现说明
下面我为你准备了两种常见环境下的实现方式，你可以直接复制参考。
#### 方式一：直接调用 HTTP API (以 cURL/Fetch 为例)
当你向 Coze 的 /v3/chat 接口发送 POST 请求时，带上 "stream": true 即可。
**HTTP 请求示例：**
```bash
curl --location --request POST 'https://api.coze.cn/v3/chat' \
--header 'Authorization: Bearer 你的_PAT_访问令牌' \
--header 'Content-Type: application/json' \
--data-raw '{
    "bot_id": "你的_BOT_ID",
    "user_id": "你的_唯一用户标识",
    "stream": true, 
    "additional_messages": [
        {
            "role": "user",
            "content": "你好，请给我讲个笑话",
            "content_type": "text"
        }
    ]
}'

```
**解析说明：**
 * 开启流式后，服务器返回的将不是普通的 JSON 对象，而是一行行的纯文本流。
 * 返回格式通常包含 event: 和 data: 两部分。例如 event: conversation.message.delta 表示正在输出消息增量，紧接着的 data: 后面会跟着一段 JSON，里面包含着具体的字符片段。你需要在代码中解析这个 JSON 并将内容追加到你的页面上。
#### 方式二：使用官方 Node.js SDK (强烈推荐)
如果你在项目中（比如用 Node.js 写的中间层）使用 Coze 的 SDK，处理起来会更加优雅。
**1. 安装 SDK:**
```bash
npm install @coze/api

```
**2. 实现代码:**
```javascript
import { CozeAPI, COZE_CN_BASE_URL } from '@coze/api';

// 1. 初始化客户端
const client = new CozeAPI({
  token: '你的_PAT_访问令牌',
  baseURL: COZE_CN_BASE_URL, 
});

async function chatWithStream() {
  // 2. 调用 chat.stream 方法发起流式请求
  const stream = await client.chat.stream({
    bot_id: '你的_BOT_ID',
    user_id: '你的_唯一用户标识',
    additional_messages: [
      {
        role: 'user',
        content: '你好，介绍一下你自己',
        content_type: 'text',
      },
    ],
  });

  // 3. 使用 for await...of 循环异步读取流式数据
  for await (const part of stream) {
    // 监听消息增量事件
    if (part.event === 'conversation.message.delta') {
      // 获取实时的字符片段并打印 (在 Vue 中，你可以把这里赋值给你的响应式 ref 变量)
      process.stdout.write(part.data.content); 
    }
  }
  
  console.log('\n--- 对话输出完成 ---');
}

chatWithStream();

```
**变量说明与调整：**
 * **token**: 你的个人访问令牌 (Personal Access Token)，可在 Coze 个人设置中生成。
 * **bot_id**: 你发布的智能体 ID，可以从 Coze 平台的 URL 中获取。
 * **part.event**: 这是你需要关注的核心事件。除了 conversation.message.delta（消息增量），流里面还会包含对话创建、完成等状态事件，你可以根据业务需求进行精细化处理。
